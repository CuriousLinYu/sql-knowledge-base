## 设计原则

- **每条规则有证据**：引用官方文档章节号
- **按版本分层**：MySQL 5.7 禁用 CTE/窗口函数，8.0 无此限制
- **深度方案独立**：复杂优化方案独立成篇，含适用场景 + 代价评估
- **纯知识，不含系统细节**：不涉及任何具体系统表名、项目代号

## 使用方式

写/审 SQL 时按需检索：
- 子查询问题 → `mysql/5.7/子查询优化.md`
- 索引失效 → `mysql/5.7/索引优化.md`
- 分页慢 → `深度方案/延迟关联.md`
- EXPLAIN 看不懂 → `EXPLAIN术语速查.md`
