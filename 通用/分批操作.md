# 批量操作分页/分批

---

## SQL-BASE-002：单次 IN 列表 ≤ 500

**严重性**：⚠️ 警告

批量操作单次 IN 列表不超过 500 条，防止锁等待和事务超时。

```java
// ✅ Guava Lists.partition 分批
List<List<Long>> batches = Lists.partition(idList, 500);
for (List<Long> batch : batches) {
    mapper.updateStatusByIds(batch);
}
```

```sql
-- ❌ 单次 5000 条，可能锁等待超时
UPDATE orders SET status = 'processed' WHERE id IN (1, 2, ..., 5000);

-- ✅ 每批 500
UPDATE orders SET status = 'processed' WHERE id IN (1, 2, ..., 500);
UPDATE orders SET status = 'processed' WHERE id IN (501, 502, ..., 1000);
```

**适用场景**：所有对数据库的批量 IN / UPDATE / DELETE / INSERT。

## SQL-BASE-003：排查 SQL 先 JOIN 头表

**严重性**：ℹ️ 提示

查询有头-行关系的表时，先从行表 JOIN 头表拿到完整上下文，不要多条 SQL 分开查再手动对。
