# 覆盖索引：避免回表查询

---

## 适用场景

- SELECT 少数列（3~6 列）
- 表百万行以上
- EXPLAIN Extra 列无 `Using index`
- QPS > 10，行数 > 10 万，写入频率 < 查询频率 50%

---

## 方案

将所有 SELECT + WHERE + ORDER BY + GROUP BY 中的列建到一个联合索引，列顺序规则：

1. **等值条件列**（WHERE col = xxx）→ 放最前面
2. **范围条件列**（WHERE col > xxx / ORDER BY col）→ 放中间
3. **SELECT 覆盖列** → 放最后

```sql
-- 原查询
SELECT user_id, status, amount, created_at
FROM orders
WHERE user_id = 1 AND status = 'pending'
ORDER BY created_at DESC;

-- 建覆盖索引
CREATE INDEX idx_cover ON orders(user_id, status, created_at, amount);

-- EXPLAIN Extra: Using index（无需回表）
```

---

## 效果

百万行表提升 50~200%。

## 常见走不到覆盖索引的情况

即使建了覆盖索引，以下情况会导致 Extra 不显示 `Using index`，仍然回表：

### 1. WHERE 条件没按索引列顺序写

```sql
-- 索引: (user_id, status, created_at, amount)

-- ✅ 走覆盖索引
SELECT user_id, status, amount FROM orders
WHERE user_id = 1 AND status = 'pending';

-- ❌ 不走覆盖索引：status 跳过了 user_id
SELECT user_id, status, amount FROM orders
WHERE status = 'pending';
```

原因：联合索引必须从第一列开始连续使用，跳过 `user_id` 直接用 `status`，索引树无法定位。

### 2. WHERE 中出现了范围查询，后续列不再走索引

```sql
-- 索引: (user_id, status, created_at, amount)

-- ✅ 等值+等值，全走索引
SELECT user_id, status, amount FROM orders
WHERE user_id = 1 AND status = 'pending';

-- ❌ user_id 范围查询后，status/created_at 不再用于索引定位
SELECT user_id, status, amount FROM orders
WHERE user_id > 100 AND status = 'pending';
-- Extra: Using index condition（ICP），不是 Using index
```

原因：范围查询后的列只用于 ICP（索引条件下推），不再参与索引树的精确定位。SELECT 中的 `amount` 如果不在索引定位列中，需要回表。

### 3. SELECT 的列不在索引中

```sql
-- 索引: (user_id, status, created_at)
-- ❌ amount 不在索引中 → 必须回表
SELECT user_id, status, amount FROM orders WHERE user_id = 1;
```

### 4. 索引列太长，一行存不下

MySQL 索引页默认 16KB。如果索引列总宽度过大（如多列 VARCHAR(1000)），一条索引记录跨页，InnoDB 可能放弃覆盖索引走回表。

### 5. 查询条件使用了函数或隐式类型转换

```sql
-- 索引: (user_id, status)
-- ❌ status 做了类型转换 → 索引失效
SELECT user_id, status FROM orders WHERE user_id = 1 AND status = 1;
-- status 是 VARCHAR，传入 INT 触发隐式转换
```

## 验证方法

```sql
EXPLAIN SELECT user_id, status, amount FROM orders
WHERE user_id = 1 AND status = 'pending';

-- 看 Extra 列：
-- Using index  → 覆盖索引生效，无回表 ✅
-- Using index condition → ICP，部分列仍需回表 ⚠️
-- Using where → 没走覆盖索引 ❌
```

## 代价

- 索引占用额外存储
- INSERT/UPDATE 需要维护索引
- SELECT 列超过 6 个时索引过大，不适用
