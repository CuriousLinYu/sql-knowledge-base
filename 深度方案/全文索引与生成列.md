# 全文索引 + 生成列：左右模糊匹配

---

## 适用场景

文本列需同时支持：
- 前缀匹配 `LIKE 'xxx%'`
- 后缀匹配 `LIKE '%xxx'`
- 中间匹配 `LIKE '%xxx%'`

**条件**：列长 ≤ 200、读 >> 写、MySQL 5.7+。

---

## 方案

```sql
-- 1. 添加生成列（反转字符串）
ALTER TABLE products ADD COLUMN name_rev VARCHAR(200)
    GENERATED ALWAYS AS (REVERSE(name)) STORED;

-- 2. 普通索引（前缀匹配）
CREATE INDEX idx_name ON products(name);

-- 3. 生成列索引（后缀匹配）
CREATE INDEX idx_name_rev ON products(name_rev);

-- 4. 全文索引（中间匹配）
ALTER TABLE products ADD FULLTEXT INDEX ft_name(name) WITH PARSER ngram;
```

---

## 匹配矩阵

| 查询类型 | SQL | 用的索引 |
|---------|-----|---------|
| 前缀 | `WHERE name LIKE 'iPhone%'` | `idx_name` |
| 后缀 | `WHERE name_rev LIKE REVERSE('phone')` | `idx_name_rev` |
| 中间 | `WHERE MATCH(name) AGAINST('phone')` | `ft_name` |
| 中文 | `WHERE MATCH(name) AGAINST('手机' IN BOOLEAN MODE)` | `ft_name` (ngram) |

---

## 代价

- 写入性能下降 5~10%（STORED 生成列需计算）
- 存储增加约 10~20%
- 全文索引有最小词长限制（默认 3/4 字符）
