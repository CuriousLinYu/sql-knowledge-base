# 生成列模拟函数索引

---

## 适用场景

MySQL 5.7 需给 JSON 字段属性或计算字段建索引（5.7 无原生函数索引）。

---

## 方案

用 `GENERATED ALWAYS AS (expr) STORED` 生成列物化计算结果，再为生成列建索引。

### 场景 1：JSON 字段建索引

```sql
-- 数据
INSERT INTO users (data) VALUES ('{"city": "Shenzhen", "age": 30}');

-- 生成列 + 索引
ALTER TABLE users ADD COLUMN city VARCHAR(100)
    GENERATED ALWAYS AS (data->>'$.city') STORED;
CREATE INDEX idx_city ON users(city);

-- 查询
SELECT * FROM users WHERE city = 'Shenzhen';
```

### 场景 2：计算字段建索引

```sql
-- 生成列：提取邮箱域名
ALTER TABLE users ADD COLUMN email_domain VARCHAR(100)
    GENERATED ALWAYS AS (SUBSTRING_INDEX(email, '@', -1)) STORED;
CREATE INDEX idx_email_domain ON users(email_domain);

-- 查询
SELECT * FROM users WHERE email_domain = 'company.com';
```

### 场景 3：大小写不敏感索引

```sql
-- 生成列：转小写
ALTER TABLE users ADD COLUMN username_lower VARCHAR(100)
    GENERATED ALWAYS AS (LOWER(username)) STORED;
CREATE INDEX idx_username_lower ON users(username_lower);

-- 查询
SELECT * FROM users WHERE username_lower = 'admin';
```

---

## VIRTUAL vs STORED

| | VIRTUAL | STORED |
|---|---|---|
| 存储 | 不占空间 | 占空间 |
| 计算 | 每次查询时计算 | 写入时计算 |
| 索引 | 8.0.13+ 可建索引 | 5.7+ 可建索引 |

**MySQL 5.7 只能用 STORED。**
