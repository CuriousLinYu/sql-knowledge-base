# 延迟关联：大分页优化

---

## 适用场景

- 必须支持"跳页"分页
- OFFSET > 10000
- SELECT 列 > 5
- ORDER BY 列有索引

---

## 方案

先在索引上分页只查主键，再用 IN 回表取完整数据。

```sql
-- ❌ 原始分页：回表 100020 次
SELECT * FROM orders ORDER BY id LIMIT 20 OFFSET 100000;

-- ✅ 延迟关联：仅 20 次回表
SELECT * FROM orders
WHERE id IN (
    SELECT id FROM orders ORDER BY id LIMIT 20 OFFSET 100000
);
```

**原理**：
1. 第 1 步：子查询只扫描 `id` 索引（覆盖索引，无回表），取出 20 个 id
2. 第 2 步：用 20 个 id 精确回表取完整行

---

## 效果对比

| | 原始分页 | 延迟关联 |
|---|---|---|
| 回表次数 | 100,020 | 20 |
| 扫描索引页 | 全部 OFFSET 行 | 全部 OFFSET 行（但无回表） |
| 适用条件 | 通用 | OFFSET > 10000 |

---

## 注意

如果 ORDER BY 列和 SELECT 列都能被一个联合索引覆盖，直接用覆盖索引即可，不需要延迟关联。
