# MySQL 5.7 版本约束

> MySQL 5.7 不支持以下语法，使用会直接报错。

---

## 1. 禁用 CTE（WITH 子句）

**严重性**：🔴 阻断

MySQL 5.7 不识别 `WITH ... AS (...)` 语法。

```sql
-- ❌ MySQL 5.7 报错
WITH recent_orders AS (
    SELECT * FROM orders WHERE created_at > '2024-01-01'
)
SELECT * FROM recent_orders;

-- ✅ 替代方案：派生表
SELECT * FROM (
    SELECT * FROM orders WHERE created_at > '2024-01-01'
) recent_orders;

-- ✅ 替代方案：临时表
CREATE TEMPORARY TABLE recent_orders AS
SELECT * FROM orders WHERE created_at > '2024-01-01';
SELECT * FROM recent_orders;
```

**证据**：MySQL 5.7 Reference Manual §13.2.15（CTE 在 8.0 引入）

---

## 2. 禁用窗口函数

**严重性**：🔴 阻断

`ROW_NUMBER() OVER()` / `RANK()` / `DENSE_RANK()` 在 MySQL 5.7 中不存在。

```sql
-- ❌ MySQL 5.7 报错
SELECT *, ROW_NUMBER() OVER (PARTITION BY dept_id ORDER BY created_at DESC) AS rn
FROM employees;

-- ✅ 替代方案：分组取 MAX(id) + JOIN
SELECT e.*
FROM employees e
INNER JOIN (
    SELECT dept_id, MAX(id) AS max_id
    FROM employees
    GROUP BY dept_id
) latest ON e.dept_id = latest.dept_id AND e.id = latest.max_id;
```

**证据**：MySQL 5.7 Reference Manual §12.21（窗口函数在 8.0 引入）

---

## 3. 大偏移量分页

MySQL 5.7 没有 `SKIP LOCKED`，大 OFFSET 分页性能差。

```sql
-- ❌ 大 OFFSET 性能差
SELECT * FROM orders ORDER BY id LIMIT 100000, 20;

-- ✅ 延迟关联方案
SELECT * FROM orders
WHERE id IN (
    SELECT id FROM orders ORDER BY id LIMIT 100000, 20
);
```

**适用条件**：OFFSET > 10000、SELECT 列 > 5、ORDER BY 列有索引。

**证据**：MySQL 5.7 性能最佳实践
